**Types (shared between server and client — use cfg to conditionally derive):**
```rust
#[derive(Clone, Debug, PartialEq, serde::Serialize, serde::Deserialize)]
pub struct AuthUser {
    pub id: i32,
    pub username: String,
}

#[derive(Debug, serde::Serialize, serde::Deserialize)]
pub struct Claims {
    pub sub: String,      // user.id.to_string()
    pub username: String,
    pub exp: usize,       // Unix timestamp (seconds)
}
```

Note: AuthUser uses `i32` (not UUID) per research — matches SERIAL id in users table. JWT `sub` stores i32 as string.

**Server functions (all gated behind `#[server]`):**

`register(email: String, password: String) -> Result<String, ServerFnError>`:
- Validate: email contains '@', password >= 8 chars. Return `ServerFnError::new(...)` on failure.
- Derive username from email local-part: `email.split('@').next().unwrap_or(&email).to_string()`.
- Hash password via `tokio::task::spawn_blocking` using argon2 `PasswordHasher` with `SaltString::generate(&mut OsRng)`.
- INSERT into users (email, username, password_hash). On unique violation for email, return `ServerFnError::new("An account with this email already exists")`. On unique violation for username, append a random 2-digit suffix and retry once.
- After successful insert, call the same JWT-signing logic as login to return a token (auto-login per CONTEXT.md).

`login(email: String, password: String) -> Result<String, ServerFnError>`:
- Query user by email. If not found, return generic error `ServerFnError::new("Invalid email or password")`.
- Verify password via `tokio::task::spawn_blocking` using argon2 `PasswordVerifier`. If mismatch, return the SAME generic error (no account enumeration).
- Build Claims with `sub: user.id.to_string()`, `username: user.username`, `exp: now + 7 days`.
- Encode JWT with `jsonwebtoken::encode` using HS256 and `EncodingKey::from_secret(JWT_SECRET.as_bytes())`.
- Read JWT_SECRET from `std::env::var("JWT_SECRET")`.
- Return the token string.

`validate_token(token: String) -> Result<AuthUser, ServerFnError>`:
- Decode JWT with `jsonwebtoken::decode::<Claims>` using `DecodingKey::from_secret` and `Validation::default()` (auto-checks exp).
- On success, return `AuthUser { id: claims.sub.parse().unwrap(), username: claims.username }`.
- On failure, return `ServerFnError::new("Invalid or expired token")`.

**Unit tests** in a `#[cfg(test)] mod tests` block at the bottom of handlers.rs. These test the logic units WITHOUT a database (no PgPool needed):

- `test_argon2_hash_and_verify`: Hash a password, verify it matches, verify a wrong password does not match.
- `test_jwt_encode_decode`: Create Claims, encode to JWT, decode back, assert fields match.
- `test_jwt_expired`: Create Claims with exp in the past, encode, attempt decode — should fail.
- `test_jwt_invalid_string`: Attempt to decode "not.a.jwt" — should fail.
- `test_username_from_email`: Test the email local-part derivation logic.

For the validation tests, use a test-local JWT_SECRET constant (e.g., `"test-secret-key"`).

Note: Integration tests requiring PgPool are out of scope for this plan — the unit tests cover the cryptographic and JWT logic in isolation.
