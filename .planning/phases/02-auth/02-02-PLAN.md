b. Add page-load JWT validation Effect (reads localStorage, calls validate_token):
```rust
Effect::new(move |_| {
    #[cfg(not(feature = "ssr"))]
    {
        let token = web_sys::window()
            .and_then(|w| w.local_storage().ok().flatten())
            .and_then(|s| s.get_item("jwt").ok().flatten());
        if let Some(token) = token {
            leptos::task::spawn_local(async move {
                if let Ok(user) = validate_token(token).await {
                    auth_user.set(Some(user));
                }
            });
        }
    }
});
```

c. Add routes for /login and /register:
```rust
<Route path=StaticSegment("login") view=LoginPage/>
<Route path=StaticSegment("register") view=RegisterPage/>
```

d. Update HomePage to show auth state + logout button:
- Read `auth_user` from context.
- If `Some(user)`: show "Welcome, {username}" and a logout button.
- If `None`: show "Not logged in" with links to /login and /register.
- Logout handler: remove "jwt" from localStorage (guarded by `#[cfg(not(feature = "ssr"))]`), set auth_user to None, navigate to "/login".

e. Update `<Title>` from "Welcome to Leptos" to "My X".

**Ensure `web_sys` import works on WASM side:** The `web_sys` crate is available when `hydrate` feature is enabled (added in Plan 01's Cargo.toml changes). The `#[cfg(not(feature = "ssr"))]` guard ensures it only runs on client.

CRITICAL: `use_navigate()` must be called inside a component that is a child of `<Router>`. Since the Effects in LoginPage, RegisterPage, and HomePage are inside components rendered by Router routes, this is safe. Do NOT call `use_navigate()` in App() directly outside the Router.
